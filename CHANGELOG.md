# Changelog

The report footer stamps the version that produced it
(`Generated by awr_trend.sql v<version>`), defined as `awr_version` at the
top of `awr_trend.sql`. Bump it there when cutting a release and add an
entry here. Dates are release dates.

## 1.1.0 — 2026-07-06

Report redesign and two table presets. Engine and read-only invariant
unchanged; everything here is client-side chrome or a rendering fix.

### Report
- **Workbench redesign** — the sticky top nav becomes a fixed left
  navigation rail: scrollspy-tracked section links, a live per-section
  status dot graded from that section's own content (crit/warn/neutral),
  panel-framed sections, and a teal accent. Falls back to a stacked layout
  under 980px.
- **Slate Instrument theme + dark mode** — a sun/moon toggle in the rail
  brand row flips light/dark. First load honours `prefers-color-scheme`;
  the choice persists in `localStorage`. Applied before charts initialize
  so there's no flash. Purely client-side; works offline.
- **"Essential rows" preset** — a rail toggle that collapses the Load
  profile, System metrics, FG/BG wait, and Findings detail tables to a
  curated shortlist of the rows a DBA scans first, with a kept/total pill
  on each affected section header. Rows flagged crit/warn stay visible even
  when off-list, so the preset never hides an anomaly; charts and the
  findings heatmap are untouched. Client-side, offline.
- Both the "Application only" and "Essential rows" toggles live in the rail
  foot (the app-only toggle moved there from the old top nav).

### Fixes
- LISTAGG positional-CSV alignment: preserve empty measure slots so
  per-week sparkline points and chart series line up with the correct
  window instead of left-compacting into the wrong one.

## 1.0.0 — 2026-07-01

First versioned release. Everything below shipped unversioned between
2026-04-20 (initial toolkit) and this date.

### Report
- 16 sections: params/nav, aligned windows, load profile, SYSMETRIC,
  FG/BG waits, Top-N SQL (5 rankings + schema/module/action breakdowns),
  z-score findings + heatmap, hero overview cards, ASH timeline, DB-time
  summary, per-SQL ASH cards, parameter changes, utilization profile,
  segment I/O, file I/O.
- "Application only" client-side toggle (hides system-wide sections and
  Oracle-internal SQL; conservative parsing-schema classification).
- Collapsible "all movers" list under the verdict.
- Timeline markers: `marker_file` (on-disk) or `markers` (inline, file-free),
  drawn on calendar-axis and per-window trend charts.
- Offline/self-contained reports: `echarts` var (CDN / internal mirror /
  local file inlined by the wrapper); pinned `vendor/echarts.min.js`
  (Apache-2.0) ships in-repo. CDN failure degrades to tables + banner.
- DB name in report filename; per-section debug progress markers (default on).

### Engine
- Read-only invariant: SELECT-only, no scratch schema, standby-safe.
- Arbitrary cadence (`step` + `step_unit`: hours/days/weeks), RAC-aware
  window validity (per-instance snap pairing, restart guard), aggregate or
  per-instance mode, additive-vs-ratio SYSMETRIC roll-up.
- Multitenant/migration-safe DBID resolution (`dbid_list` spanning
  non-CDB → PDB migrations); NLS-proof numeric round-trips.
- Templates: `comprehensive` (default), `simple`, `dev`.

### Tooling
- `run_awr_trend.sh` wrapper + interactive configurator (`--configure`);
  symmetric pure-SQL*Plus path (`sql/defaults.sql` + `awr_trend.sql`).
- Optional `side/create_weekly_baselines.sql` (the only writer).
- GitHub Pages site under `docs/` with example reports and cheatsheet.
